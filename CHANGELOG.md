# 更新日志

本文档记录每次功能添加、修改、删除的内容。后续开发必须在同一次变更中同步更新本文件。

## 2026-05-09

### 添加

- 初始化 Python / uv 项目结构，加入 `openai`、`pillow`、`pyside6`、`python-dotenv`、`tzdata` 等运行依赖。
- 添加屏幕截图与图片编码能力，支持 Windows 桌面截图、图片缩放、JPEG/PNG data URL 输出。
- 添加 GPT 兼容视觉模型调用，支持 `OPENAI_API_KEY`、API Base URL、模型名配置，并要求返回结构化 JSON。
- 添加活动识别 Prompt，包含分类体系、视频分类规则、隐私脱敏规则、未知处理规则和禁止评价用户行为规则。
- 添加最近 5 条活动记忆，用于辅助判断当前活动是否延续。
- 添加本地数据存储：原始记录写入 `data/raw/YYYY-MM-DD.jsonl`，合并事件写入 `data/events/YYYY-MM-DD.json`。
- 添加连续活动合并逻辑，按分类、事件相似度、时间间隔和 `is_continuation` 判断是否合并。
- 添加命令行单次识别入口 `screen-agent-once`。
- 添加 API 诊断入口 `screen-agent-diagnose`，用于测试文本调用、图片调用和截图图片调用。
- 添加 PySide6 图形界面，包含控制台、日志、配置三个页面。
- 添加控制台操作：开始记录、暂停、手动识别一次、查看最近识别结果。
- 添加日志页面，支持按日期查看合并事件和原始识别记录。
- 添加配置页面，支持编辑 API Key、API Base URL、模型名和截图间隔。
- 添加配置完整性检查：如果 API Key、URL 或模型未配置，启动后自动进入配置页面。
- 添加多套 API 配置管理：保存配置、应用配置、删除配置。
- 添加 `api_profiles.json`，用于保存多套 API 配置档案。
- 添加 Windows PowerShell 启动脚本 `run_gui.ps1`、`run_once.ps1`、`view_today.ps1`。
- 添加 PyInstaller 打包脚本 `build_exe.ps1`。
- 生成 Windows 图形界面可执行文件 `dist/ScreenActivityAgent/ScreenActivityAgent.exe`。
- 添加分类与隐私兜底模块，统一规范一级分类、视频内容分类和敏感内容识别。
- 添加代码级隐私脱敏，敏感页面只保留概括信息，不保存敏感原文。
- 添加报表模块 `screen-agent-report`，支持日报、周报和月报文本输出。
- 添加 GUI 统计页，支持按日期切换查看日 / 周 / 月统计。
- 添加滚动统计周期 `last7` 和 `last30`，支持查询所选日期向前 7 天或 30 天的活动统计。
- 首页新增“今日已记录时长”和“今日主要分类占比”，打开 GUI 后可以直接看到当天记录总量和主要分类分布。
- 新增独立“时间线”页面，支持按日期切换、按分类筛选并展示所选日期的完整合并事件时间线。
- 设置页新增数据保存目录、是否保存原始截图、隐私保护开关、敏感内容过滤开关、开机自启配置，并写入 `.env`。
- 记录管理页新增按分类筛选，合并事件和原始识别记录会按所选日期与分类过滤展示。
- 记录管理页改为表格管理界面，支持查看记录详情、编辑合并事件、删除选中记录、导出 JSON 和导出 CSV。
- 新增 `scripts/rebuild_events.py`，可以从原始 JSONL 重建指定日期的合并事件 JSON，并默认备份旧事件文件。
- 阶段九美化 GUI：新增统一浅色工作台样式、摘要卡片、分区面板和分类占比条形图。
- 首页新增可视化分类占比图，并将运行状态、今日时长、主要分类和事件数量做成摘要卡片。
- 统计页新增总时长、最高分类、未知占比摘要卡片和分类占比图表。
- 时间线页新增“时间分布”图表，用横向时间轴展示当天活动段、空档和主要分类颜色，表格保留用于查看精确记录。
- GUI 图表配色改为固定一级分类颜色，避免娱乐、编程等常见分类因哈希配色过于接近而难以分辨。
- 配置页将“保存设置”改为“保存运行设置”并移到运行设置区域；按钮仅在运行设置发生改动后启用，API 配置的应用和修改会立即生效，不再混用运行设置保存逻辑。
- 修复“保存运行设置”禁用后仍显示为蓝色的问题，并规范数据目录路径比较，避免运行设置改动状态误判。

### 修改

- 将默认运行环境明确为 Windows 本地桌面。
- 将默认 API Base URL 规范化为 `https://apiport.cc.cd/v1`。
- 将默认模型调整为 `gpt-5.5`，同时保留通过环境变量覆盖的能力。
- 优化截图请求体，默认使用 JPEG、宽度压缩和单屏截图，降低模型网关拦截概率。
- 配置读取改为自动查找 `.env`，便于 exe 目录和项目根目录之间复用配置。
- GUI 状态语义改为“已暂停 / 运行中”，关闭窗口时会停止后台 worker。
- 控制台页移除“修改配置”按钮，配置切换统一通过顶部“配置”页进入。
- 配置页改为显示当前应用配置名称，并通过“添加配置”子页面新增 API 配置。
- 配置列表新增“修改”按钮，支持在原配置基础上编辑名称、API Key、URL 和模型，并允许名称保持为当前配置名。
- 配置列表改为每行直接显示“应用 / 已应用”状态，当前应用配置使用绿色“已应用”按钮区分。
- 模型 Prompt 与数据层现在都执行分类与隐私规则，减少对单纯 Prompt 的依赖。
- 新增报表聚合逻辑，支持按日、周、月统计一级分类占比、二级分类 Top 活动和每日趋势。
- GUI 统计页和 CLI 报表入口都支持滚动周期；滚动周期不按自然周或自然月切分。
- Windows 打包从全量收集 PySide6 改为只收集实际需要的模块和 `tzdata`，减少构建时间和产物体积。
- README 补充 Windows 运行、诊断、打包、多配置和日志查看说明。
- 日志与记录读取工具统一使用 UTF-8 JSON/JSONL，并提供共享的分类标签、时长格式化、筛选、保存和导出逻辑。
- 开机自启在开发运行模式下会先切换到项目目录再启动 GUI，避免 Windows 登录启动时找不到 Python 模块。
- 时间线、记录管理、日志、配置等子页面改为统一的标题、工具栏和内容面板布局，提升扫描和操作效率。
- 记录管理表格调整为无网格线、交替行底色和更清晰的表头样式。
- 恢复中文 `README.md` 作为 GitHub 仓库首页，集中介绍项目用途、下载运行、首次配置、隐私说明和源码构建方式。
- 保留中文 `RELEASE.md` 作为发布说明，面向下载 release 的普通用户。
- README 末尾新增中英文项目声明，包含项目名称、作者和作者单位。
- RELEASE 下载说明新增 release 页面链接，方便用户直接跳转下载 Windows 压缩包。

### 修复

- 修复手动识别和持续识别时用户 Prompt 引用未定义变量 `primary_categories` 导致“识别失败”的问题。
- 修复持续记录时长偏低的问题：现在每次新识别会先把上一段活动延长到当前识别时间，再决定合并或开启新事件，避免连续运行时大量记录只计为 0-1 分钟。
- 修复统计页分类占比图右侧时长和百分比显示空间不足导致文字被截断的问题。
- 修复记录管理表格在开启排序后填充数据导致标题与数据错位、部分列显示缺失的问题。
- 修改 API 配置弹窗的 API Key 展示方式，改为前后掩码显示并提供复制完整 API Key 的按钮。
- 修复时间线页面记录变多后反复重建大段文本导致 GUI 卡顿、后续识别结果无法及时回写的问题；时间线改为滚动表格，并且识别完成后只刷新首页摘要和当前页面。
- 修复首页分类占比图中较长时长文本显示不完整的问题，扩大时长列并调整百分比列间距。
- 修复 API 网关无响应时开始记录长期卡住、暂停反馈不明确以及识别中切换配置可能导致闪退的问题；视觉请求现在有超时，识别中会阻止切换配置并显示明确提示。
- 修复 API 请求阻塞初期暂停按钮没有等待提示、OpenAI SDK 自动重试导致超时反馈过慢的问题；开始识别会立即进入请求中状态，并关闭 SDK 自动重试。

### 删除

- 移除“只能通过命令行查看记录”的限制，改为 GUI 日志页可直接查看。
- 打包流程不再全量收集 PySide6 所有模块，避免构建过慢和产物过大。
- 不再依赖模型单独决定隐私页面的保留方式，代码会在落盘前进行脱敏处理。
- 移除英文 release 文档内容，改为中文说明。
- 移除 README 末尾的中英文项目声明。
